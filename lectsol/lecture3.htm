    6. Write bash functions marco and polo that do the following. Whenever you execute marco the current working directory should be saved in some manner, then when you execute polo, no matter what directory you are in, polo should cd you back to the directory where you executed marco. For ease of debugging you can write the code in a file marco.sh and (re)load the definitions to your shell by executing source marco.sh.

      marco_polo.sh

      #!/usr/bin/bash

function marco() {
	#Save current directory in global variable
	export MARCO_DIR=$(pwd)
}

function polo() {
	#Verify if there is a saved directory
	if [ -z "$MARCO_DIR" ]; then
		echo "No directories saved, please run 'marco' first."

	else
		#Change directory and display error message if failed
		cd $MARCO_DIR || echo "Failed to change directory to $MARCO_DIR"
	
	fi
}

      Marco assigns the current directory to a avariable using export to ensure the variable has environment scope.

      Polo first verifies if there is a directory stored (-z verifies if a value is empty).

      It then changes the directory and displays a message if it fails with ||

    8. As we covered in the lecture find’s -exec can be very powerful for performing operations over the files we are searching for. However, what if we want to do something with all the files, like creating a zip file? As you have seen so far commands will take input from both arguments and STDIN. When piping commands, we are connecting STDOUT to STDIN, but some commands like tar take inputs from arguments. To bridge this disconnect there’s the xargs command which will execute a command using STDIN as arguments. For example ls | xargs rm will delete the files in the current directory.

      Your task is to write a command that recursively finds all HTML files in the folder and makes a zip with them. Note that your command should work even if the files have spaces (hint: check -d flag for xargs).

      If you’re on macOS, note that the default BSD find is different from the one included in GNU coreutils. You can use -print0 on find and the -0 flag on xargs. As a macOS user, you should be aware that command-line utilities shipped with macOS may differ from the GNU counterparts; you can install the GNU versions if you like by using brew.

      1. First you need to install zip if it isn't already done.
        sudo apt install zip
      2. Then you can run
        find . -name "*.html" -print0 | xargs -0 zip html_files.zip

        find . searches the current directory

        *.html matches any file ending with .html

        -print0 and -0 arguments handle whitespace in file names by seperating files with null characters which won't interfere with whitespace in file names

    9. (Advanced) Write a command or script to recursively find the most recently modified file in a directory. More generally, can you list all files by recency?
      find . -type f -exec stat --format="%Y %n" {} + | sort -n -r | head -n1 | sed 's/^[0-9]* //'

      find . -type f -exec stat --format="%Y %n" {} +: Finds all regular files recursively and prints the modification time followed by the file name.

      sort -n -r: Sorts the files by modification time.

      head -n1: Selects the first (most recently modified) file.

      sed 's/^[0-9]* //': Uses sed to remove the number (modification time in seconds) at the beginning of the line. The ^[0-9]* matches the leading digits, and the space following removes the space between the time and the file name. You will learn more about sed in the upcomming lectures.
